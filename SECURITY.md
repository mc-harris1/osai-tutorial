# Security Policy

## Supported Versions

This repository is a documentation and educational project. Security updates are applied to the latest version available on the `main` branch.

## Reporting a Security Issue

If you discover a security issue related to this repository (for example, a vulnerable script or an accidental disclosure of sensitive information), please do **not** open a public GitHub issue.

Instead:

1. Contact the maintainer privately.
2. Include a description of the issue.
3. Provide reproduction steps if applicable.
4. Include any suggested remediation.

The maintainer will acknowledge receipt, investigate the issue, and provide updates as appropriate.

## Scope

This policy applies to:

* Repository scripts
* GitHub Actions workflows
* Documentation that could expose sensitive information
* Sample configurations

This policy does **not** apply to security vulnerabilities in third-party software such as NVIDIA DGX Spark hardware, Linux distributions, Mellanox/ConnectX drivers, Wireshark, or other vendor products. Those should be reported to the appropriate vendor.

## Responsible Disclosure

Please do not publicly disclose a vulnerability until it has been evaluated and, when appropriate, remediated.
