# Contributing

Thank you for your interest in improving OSaI.

## Ways to Contribute

* Improve documentation
* Correct errors
* Report bugs
* Suggest new labs
* Improve scripts
* Add diagrams
* Improve benchmarking methodology

## Before Opening an Issue

Please search existing issues to avoid duplicates.

## Pull Requests

Pull requests should:

* Have a clear purpose.
* Be focused on a single change.
* Include documentation updates when appropriate.
* Follow existing Markdown formatting.
* Keep examples reproducible.

## Documentation

When contributing tutorials:

* Explain the objective.
* Describe prerequisites.
* Include expected results.
* Document any assumptions.
* Prefer reproducible examples over screenshots alone.

## Code

Scripts should:

* Be well documented.
* Favor portability.
* Avoid unnecessary dependencies.
* Fail safely with meaningful error messages.

## Questions

If you are unsure about a proposed change, open a discussion or question before investing significant effort.

Thank you for helping improve the project.
